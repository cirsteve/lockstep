# Lockstep — Hackathon Pitch

*For hackathon judges, sponsor team evaluations, and audiences that need to see the demoable artifact first and the platform thesis as closing argument.*

---

**The first strategy marketplace where the producer doesn't choose the test.**

Quants mint trading strategies as ERC-7857 iNFTs on 0G Chain. The encrypted strategy lives on 0G Storage; a signed evaluation receipt anchors the token.

That receipt is generated by an open-source grader running inside a TEE on 0G Compute. The grader ingests the strategy and a marketplace-published canonical dataset — Merkle-committed BTC/ETH/SOL hourly across labeled regimes, with a sealed private holdout the producer never sees. It runs walk-forward evaluation, computes per-regime Sharpe and drawdown, and signs the result with an enclave-attested key.

The producer optimizes their strategy. They never touch the evaluation data, the scoring environment, or the holdout set. Self-selected tests and cherry-picked backtests — the failure mode of every existing strategy marketplace — are structurally constrained, not just discouraged.

The enclave produces the receipt. The validator set verifies reproducibility. Anyone can run the OSS grader as an AXL node, randomly sample graded iNFTs, and re-run evaluation against the public dataset Merkle root. Divergence from the attested receipt can be challenged on-chain.

Trust the enclave for execution integrity. Trust the validator set for reproducibility.

Grader versions are immutable and content-addressed. New versions publish updated dataset roots and scoring rules; existing receipts remain verifiable under their original grader.

Allocators browse by worst-regime performance, not aggregate Sharpe. They pay producers via x402 and call `authorizeUsage()` to rent the strategy. KeeperHub is the sealed executor — running signals through Turnkey-secured non-custodial wallets, landing trades on Aave, Spark, or Morpho with full retry logic.

Strategy code stays encrypted end-to-end. Allocators receive execution outcomes, not the strategy itself.

Every live execution flows back as new evaluation data. Rankings shift from backtest-weighted to live-weighted as track records accumulate. Backtests get you on the leaderboard. Live performance keeps you there.

Producers monetize alpha without disclosure. Allocators get standardized, reproducible evaluation instead of unverifiable backtests.

The grader defines the rules. The enclave enforces them. The validator network keeps them honest. Live performance has the final word.

We don't certify alpha. We make cheating the test harder than finding real signal.

---

**Trading is the wedge. The substrate is the product.**

What we've actually built is a protocol for verifiable evaluation of private computation. Three primitives — attested grading, sampling validation, sealed execution — composed into a substrate that solves the trust problem gating every market for valuable computation.

The same primitives generalize beyond trading. ML model benchmarks where solvers submit models and the canonical eval is a sealed test set. Optimization solutions where problem-posters lock bounties in escrow and solvers compete. Forecasting tournaments where time provides natural holdout. Code and security artifacts where graders run fuzzing harnesses against canonical vulnerability databases.

Each new domain is a Layer 3 instantiation on top of the same substrate. We ship two trading instantiations in this release — directional perp strategies and market-neutral funding-rate arbitrage — explicitly to prove the abstraction holds across meaningfully different evaluation shapes.

We chose trading because it has high value, clear metrics, existing supply of underutilized quant talent, and clean composability with KeeperHub's existing sealed-executor infrastructure. Every sponsor's technology in this submission does work the others can't, and the substrate cannot exist without all three: 0G's chain, storage, and compute for attested grading and rentable assets; KeeperHub's workflows for sealed execution and reliable settlement; Gensyn's AXL for the permissionless validator transport.
